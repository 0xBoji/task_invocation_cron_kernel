# TICK v0.2 Design

## Goal
Extend `tick` with explicit job payload modes (`wasm`, `shell`), explicit execution policies (`local_only`, `mesh_any`, `remote_only`), direct CAMP crate integration, and local WASP-backed dispatch while keeping remote execution simulated and future-ready.

## Architecture
- Scheduler triggers persisted cron jobs.
- Mesh provider queries CAMP directly through `coding_agent_mesh_presence`.
- Selection filters by role + idle status, applies execution policy, and load-balances via round-robin.
- Local selections run through `LocalDispatcher`.
- Remote selections emit `remote_dispatch_simulated` and count as orchestration success.

## Data model
- `ExecutionPolicy::{LocalOnly, MeshAny, RemoteOnly}`
- `JobType::{Wasm, Shell}` with `Wasm` as the CLI default
- Persist jobs to a zero-config JSON file under the tick data dir.

## Testing
- Mock `MeshProvider` and `Dispatcher` in unit tests.
- Do not spawn `wasp` or open real CAMP networking in unit tests.
- Verify JSON daemon events always include `rai_component` and `rai_level`.
